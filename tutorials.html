Beginner's Tutorials/Guides for PHP

Below is a selection of tutorials or guides that can be used to become proficient in using PHP.

Follow these in order:

  1. http://php.net/manual/en/tutorial.php
  2. https://www.phptherightway.com/
  3. https://github.com/PatrickLouys/no-framework-tutorial (Note: This is slightly outdated in some parts, but it is still a good, practical tutorial in learning how to write professional PHP)
  4. https://patricklouys.com/professional-php/ This is a book written by the same author for the same No-Framework Tutorial, but is updated.

Once you have familiarity with those, you can delve a little deeper...

  • https://stackoverflow.com/questions/16356420/mvc-for-advanced-php-developers - a list of videos and reading material
  • https://stackoverflow.com/questions/134099/are-pdo-prepared-statements-sufficient-to-prevent-sql-injection
  • http://wiki.hashphp.org/PDO_Tutorial_for_MySQL_Developers - learning how to use PDO when coming from a MySQL/MySQLi background
  • https://phpdelusions.net/pdo - alternatively, this is a thorough guide on learning PDO
    • - using prepared statements in PDO is of the utmost importance, but they're not completely infallible, this explains how to further protect your queries while using prepared statements
  • https://stackoverflow.com/questions/5863870/how-should-a-model-be-structured-in-mvc - an explanation of what models are in the MVC relationship
  • https://paragonie.com/blog/2017/12/2018-guide-building-secure-php-software - a newer guide on writing secure PHP software
  • http://php.net/manual/en/faq.passwords.php - explanation on password hashing

PHP isn't the only thing you should learn! A useful skill is knowing how to set up your own development environment without using XAMPP. Below are guides on setting up different development environments. (Note that VM means Virtual Machine)

https://www.vagrantup.com/intro/getting-started/index.html - Vagrant is a software that allows you to set up a development environment VM quickly. You can find vagrantboxes that other people have created and uploaded, in which you can use instead of setting up a VM in Vagrant from scratch

https://docs.docker.com/get-started/ - another common development environment is Docker. It is an alternative to VMs because it does not require the setting up and maintenance that regular VMs do. It is able to do this through the use of containers.

The options below are guides on setting up a development environment from scratch using a VM on your computer.

  • Setting up a LAMP environment (Linux [Ubuntu], Apache, MySQL, PHP)
  • Setting up a LEMP environment (Linux [Ubuntu], nginx, MySQL, PHP)
  • Another setting up a LEMP environment (Linux [Arch], nginx, MySQL, PHP

If you have an aversion to Oracle products, you can use mariaDB which is virtually the same as MySQL, except that it is not owned by Oracle.

Lastly, this guide explains how to set up shared folders in Virtualbox from Windows. It took me a lot of digging to find it, and it was extremely helpful for me.
